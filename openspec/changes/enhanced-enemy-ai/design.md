## Context

儿童游戏平台中的敌人系统目前只实现了基础的几何体模型，缺乏完整的AI行为体系。为提升游戏体验，需要实现更真实的敌人行为系统。当前系统基于Three.js实现，需要在不改变现有架构的前提下扩展敌人AI功能。

## Goals / Non-Goals

**Goals:**
- 实现完整的敌人骨骼动画系统
- 实现敌人巡逻路径和行为管理
- 实现敌人状态机和决策逻辑
- 实现敌人头顶血条和实时血量显示
- 实现击中反馈和死亡动画效果
- 保持与现有项目架构的一致性

**Non-Goals:**
- 实现复杂的物理引擎或AI算法
- 为敌人添加语音或声音效果
- 改变现有的3D渲染架构

## Decisions

### 1. 动画系统实现方案

**选择 Three.js 动画系统**
- 原因：比手动实现更稳定可靠，与现有Three.js集成良好
- 替代方案：手工实现动画关键帧 - 复杂度高且易出错

### 2. 敌人状态管理

**选择有限状态机模式**
- 原因：适合敌人行为的逻辑简单且可预测的场景
- 替代方案：行为树 - 过于复杂且当前需求不支持

### 3. 血条显示系统

**选择3D空间投影到2D UI方案**
- 原因：提供最佳视觉效果，符合游戏设计理念
- 替代方案：使用固定像素位置 - 交互性差

## Risks / Trade-offs

- **性能风险** → 优化动画和渲染逻辑，避免帧率下降
- **复杂度提升** → 保持模块化设计，增加代码可维护性
- **内存占用增加** → 使用对象池优化敌人实例复用

## Migration Plan

1. 首先实现基础敌人动画和状态机
2. 渐进式添加血条系统
3. 逐步完善击中反馈和死亡动画
4. 进行集成测试保证性能
5. 修复可能引入的兼容性问题
