## ADDED Requirements

### Requirement: 玩家死亡判定

当玩家血量归零时，触发死亡流程。

#### Scenario: 玩家血量归零触发死亡
- **WHEN** 玩家血量减少至 0 或以下
- **THEN** 游戏状态切换为 `ended`
- **AND** 触发死亡界面显示

### Requirement: 死亡界面显示

玩家死亡后显示游戏统计和重生选项。

#### Scenario: 显示死亡界面
- **WHEN** 玩家死亡
- **THEN** 游戏画面上方覆盖半透明黑色遮罩（`rgba(0, 0, 0, 0.8)`）
- **AND** 显示 "你已阵亡" 标题
- **AND** 显示存活时间和击杀数统计

#### Scenario: 死亡界面显示统计信息
- **WHEN** 死亡界面显示时
- **THEN** 显示玩家存活时间（分:秒格式）
- **AND** 显示玩家击杀敌人数量
- **AND** 显示玩家获得的总分数

### Requirement: 玩家重生

玩家可以通过按钮选择重新开始游戏。

#### Scenario: 点击重新开始按钮
- **WHEN** 玩家点击死亡界面中的 "重新开始" 按钮
- **THEN** 重置玩家血量为 100
- **AND** 重置游戏分数和击杀数为 0
- **AND** 重新生成所有敌人
- **AND** 游戏状态切换为 `playing`

#### Scenario: 点击返回主页按钮
- **WHEN** 玩家点击死亡界面中的 "返回主页" 按钮
- **THEN** 退出游戏 Pointer Lock
- **AND** 跳转到游戏首页

### Requirement: 死亡界面视觉设计

死亡界面采用 Q 版卡通风格，符合项目整体视觉定位。

#### Scenario: 死亡界面样式
- **WHEN** 死亡界面显示时
- **THEN** 标题使用较大字体居中显示
- **AND** 统计信息使用卡片式布局
- **AND** 按钮使用圆角矩形，悬停时有放大效果
