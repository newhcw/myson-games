## Context

当前敌人系统存在的问题：

1. **攻击范围过小**：现有攻击判定距离仅 1.5 单位，敌人难以命中玩家
2. **近战机制不适合 3D FPS**：玩家可以远距离风筝敌人，敌人无法有效反击
3. **缺乏伤害反馈**：玩家不知道自己被击中，缺乏紧张感
4. **无死亡机制**：玩家血量归零后仅结束游戏，无死亡动画和重生

项目约束：
- 纯前端实现，无后端依赖
- 使用 Vue 3 + Three.js 技术栈
- 目标用户是儿童，视觉风格 Q 版卡通化

## Goals / Non-Goals

**Goals:**
- 实现敌人远程射击能力，弥补当前游戏缺乏挑战性的问题
- 提供清晰的伤害反馈，让玩家感知到威胁
- 实现死亡和重生机制，增强游戏完整性

**Non-Goals:**
- 不实现敌人躲避障碍物（后续可优化）
- 不实现敌人之间的协作战术
- 不实现武器升级或弹道下坠等高级射击机制

## Decisions

### Decision 1: 敌人射击采用射线检测而非弹幕

**选择**：使用 Three.js Raycaster 进行射线检测

**理由**：
- 与玩家射击系统一致，保持代码一致性
- 性能高效，适合少量敌人
- 可精确控制射击方向和精度

**替代方案考虑**：
- 抛射物检测：创建子弹实体并检测碰撞 → 性能开销大，不需要
- 范围伤害：范围内直接扣血 → 过于简单，缺乏技巧性

### Decision 2: 射击精度采用固定随机偏移

**选择**：每次射击时，方向在 ±10度 范围内随机偏移

**理由**：
- 模拟真实射击的不确定性
- 玩家可以通过走位躲避部分子弹
- 实现简单，参数可调

**参数配置**：
```typescript
const SHOOT_CONFIG = {
  attackInterval: 1000,    // 射击间隔 1 秒
  accuracyOffset: 10,       // 精度偏移 ±10度
  viewDistance: 15,         // 射程 15 单位
  minViewAngle: 30,         // 有效射击角度 30度
}
```

### Decision 3: 伤害反馈使用 CSS 遮罩层

**选择**：屏幕边缘红色渐变遮罩 + 血条平滑动画

**理由**：
- CSS 实现简单，性能好
- 可通过 opacity 过渡实现渐变效果
- 不影响游戏性能

**效果参数**：
```typescript
const DAMAGE_FEEDBACK_CONFIG = {
  redOverlayColor: 'rgba(255, 0, 0, 0.3)',  // 屏幕闪红透明度
  fadeOutDuration: 500,                      // 渐变消失时间 500ms
  healthTransitionDuration: 300,            // 血条过渡时间 300ms
}
```

### Decision 4: 死亡界面使用 Vue 组件实现

**选择**：独立的 DeathScreen.vue 组件覆盖游戏画面

**理由**：
- 组件化设计，职责清晰
- 便于添加动画和交互
- 与游戏主循环解耦

**界面元素**：
- 半透明黑色遮罩背景
- "你已阵亡" 标题
- 存活时间、击杀数等统计
- "重新开始" 和 "返回主页" 按钮

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 敌人射击过难导致玩家挫败 | 儿童玩家可能失去兴趣 | 默认难度较低，可配置难度参数 |
| 射线检测穿墙问题 | 敌人可能攻击到墙后的玩家 | 可在射线检测时检查障碍物遮挡 |
| 死亡界面遮挡导致无法操作 | 按钮点击无响应 | 使用 pointer-events: auto 确保按钮可点击 |

## Open Questions

1. **射击音效**：是否需要为敌人射击添加独特的音效？
2. **射击特效**：敌人射击时是否需要可见的弹道/光线效果？
3. **难度曲线**：随着时间推移或分数增加，是否需要增加敌人伤害或射速？
