## Context

儿童游戏平台，从零构建。需要纯 H5 实现 3D FPS 射击游戏，Vue 3 + Vite 前端，所有数据本地存储。核心约束：目标用户是儿童，视觉风格必须 Q 版卡通化，无后端服务器依赖。

## Goals / Non-Goals

**Goals:**
- 实现一个可扩展的游戏平台框架（首页可展示多个游戏入口）
- 实现 3D 第一人称射击游戏，适合儿童（Q 版敌人、卡通场景）
- 支持武器系统（多种武器、倍镜切换）
- 支持敌人 AI（巡逻走动、被击中显示血量、死亡动画）
- 支持玩家自定义（角色形象、武器、倍镜）
- 全部数据本地持久化（配置、进度、成就）
- 纯 H5 运行，无需安装插件，浏览器直接打开
- 实现儿童友好的 UI 主题（色彩鲜明、字体清晰、动画有趣）

**Non-Goals:**
- 不做多人在线/联机功能（首期单人对战 AI 敌人）
- 不做后端服务器（全部本地存储）
- 不做移动端原生 APP（纯 H5 网页）
- 不做内购/付费系统
- 不做反作弊系统（单人游戏无此需求）
- 不做真实的暴力/血腥效果（儿童友好）
- 不做移动端触控优化（首期仅支持键鼠操作）

## Decisions

### 1. 3D 渲染引擎：Three.js

| 方案 | 理由 |
|------|------|
| **Three.js（选择）** | 生态最大，社区活跃，Vue 集成方便（@tresjs/cientos），文档丰富 |
| Babylon.js | 微软维护，功能强大但社区较小，学习曲线陡峭 |
| PlayCanvas | 性能好但需要在线编辑器，不适合本地开发 |

选择 Three.js + TresJS（@tresjs/core）方案：
- 可直接在 Vue 组件中使用声明式 3D 场景
- 支持 Vue 3 Composition API
- 良好的 TypeScript 支持

### 3. 状态管理：Pinia

游戏内状态使用 Pinia store 统一管理：

| Store | 职责 |
|-------|------|
| `game.js` | 游戏状态（开始/暂停/结束）、当前分数、游戏时间 |
| `player.js` | 玩家血量、当前位置、角色形象选择 |
| `weapon.js` | 当前武器、弹药数量、倍镜配置、武器列表 |
| `settings.js` | 音量设置、画质设置、控制键位配置 |

### 4. 本地存储：IndexedDB + localStorage

| 存储方式 | 用途 | 数据类型 |
|----------|------|----------|
| **localStorage** | 轻量配置数据 | 游戏设置、音量、最高分、解锁状态 |
| **IndexedDB** | 大量结构化数据 | 玩家自定义配置、多游戏存档、成就记录 |

封装统一的 `useStorage` composable，对外提供一致的 API：
- `saveGameProgress(progress)` - 保存游戏进度
- `loadGameProgress()` - 加载游戏进度
- `saveSettings(settings)` - 保存设置
- `loadSettings()` - 加载设置
- `saveHighScore(gameId, score)` - 保存最高分
- `getHighScores()` - 获取排行榜

### 5. Q 版敌人设计

3D 模型统一使用 glTF/GLB 格式（Web 标准），通过 Three.js GLTFLoader 加载。

**敌人模型要求：**
- Q 版比例（头身比约 1:1 或 1:1.5）
- 卡通材质（Toon Shading 或 Unlit）
- 基础骨骼动画（待机、走动、死亡、受击）

**敌人类型设计：**

| 类型 | 血量 | 移动速度 | 特点 |
|------|------|----------|------|
| 小兵 | 100 | 中等 | 基础敌人，最常见 |
| 精英 | 200 | 快速 | 会躲避，更聪明 |
| BOSS | 500 | 慢速 | 大型敌人，高伤害 |

### 6. 碰撞检测：Three.js Raycasting

射击使用射线检测而非物理引擎子弹。

**工作原理：**
1. 玩家点击开火
2. 从相机位置沿视角方向发射射线
3. 检测射线与敌人碰撞体的交点
4. 计算伤害并触发反馈

**优点：**
- 即时命中，适合 FPS 场景
- 性能优于物理模拟子弹
- 实现简单，易于调试

### 7. 敌人 AI：状态机 + Waypoint 系统

```
┌─────────┐  到达目标  ┌──────────┐
│  Patrol │───────────▶│  Wait    │
│  巡逻中 │            │  等待中  │
└────┬────┘◀───────────└────┬─────┘
     │  等待结束             │  巡逻到下一个点
     │                      │
     └──── 发现玩家 ────────▶│
                            ▼
                      ┌──────────┐
                      │  Chase   │
                      │  追逐中  │
                      └────┬─────┘
                           │ 丢失玩家
                           ▼
                      ┌──────────┐
                      │  Search  │
                      │  搜索中  │
                      └──────────┘
```

**AI 状态说明：**
- **Patrol（巡逻）**：在预设路径点间来回移动
- **Wait（等待）**：到达路径点后短暂停留
- **Chase（追逐）**：发现玩家后追踪攻击
- **Search（搜索）**：丢失目标后搜索最后已知位置

### 8. 武器系统设计

| 武器类型 | 伤害 | 射速 | 弹夹容量 | 特点 |
|----------|------|------|----------|------|
| 手枪 | 25 | 慢 | 12 | 初始武器，精准度高 |
| 步枪 | 35 | 中等 | 30 | 全能型，适合中距离 |
| 冲锋枪 | 20 | 快 | 50 | 近距离高输出 |
| 狙击枪 | 100 | 很慢 | 5 | 高伤害，支持倍镜 |

**倍镜系统：**
| 倍镜 | 放大倍率 | 适用武器 |
|------|----------|----------|
| 红点 | 1.5x | 全部武器 |
| 全息 | 2x | 步枪、狙击枪 |
| 4倍镜 | 4x | 狙击枪专用 |

### 9. 儿童友好 UI 设计

**色彩方案：**
- 主色调：明亮的蓝色系 (#4A90D9)
- 强调色：橙色 (#FF9500)
- 背景：浅色渐变，柔和过渡
- 危险提示：使用绿色/黄色，避免红色警示

**字体规范：**
- 标题：圆润的无衬线字体
- 正文字号：≥ 16px，确保可读性
- 按钮：大尺寸，易于点击

**动画效果：**
- 按钮悬停：放大 + 发光效果
- 得分提示：弹出动画 + 数字飞升
- 击杀反馈：卡通星星/烟花特效
- 界面切换：平滑淡入淡出

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| Three.js 在低端设备性能不足 | 限制场景面数、使用 LOD（细节层级）、提供低画质选项 |
| 3D 模型资源获取困难 | 使用免费开源模型（Mixamo、Sketchfab）或简化为低多边形风格 |
| 移动端 H5 3D 性能差 | 首期只保证桌面浏览器流畅，移动端后续优化 |
| 本地存储可被用户清空 | 不依赖关键进度，所有数据可重建；提供导出/导入功能 |
| 儿童沉迷风险 | 内置游戏时长提醒（每 30 分钟弹窗提醒休息） |
| 音效版权问题 | 使用免费商用音效资源（Freesound.org）或自行制作 |

## Open Questions

1. **敌人模型来源**：使用免费开源模型还是委托制作？需要在开发初期确定
2. **地图数量**：首期实现几个游戏地图？建议 1-2 个以保证质量
3. **武器数量**：首期需要多少种武器？建议 3-4 种基础武器
4. **音效方案**：是否使用 Web Audio API 直接播放，还是需要更复杂的音频管理？
5. **难度曲线**：如何设计适合儿童的难度曲线？是否需要多个难度级别？
