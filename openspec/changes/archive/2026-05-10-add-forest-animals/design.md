## Context

当前 `EnvironmentManager` 管理太阳（Sun）和小鸟（Bird），两者都在高空飞行。地面层缺乏装饰元素。需要增加地面动物来丰富森林主题。

**现有模式参考**：Bird 类使用 Three.js 几何体组合 + 圆形轨道飞行 + 翅膀扇动动画。动物将沿用同样的几何体组合模式，行走在地面层。

**场景布局**：以玩家为中心 ±15 单位范围有各类树木、岩石、灌木等障碍物。动物活动范围在半径 6~20 的外环区域，避免与战斗区域重叠。

## Goals / Non-Goals

**Goals:**
- 创建 4 种卡通动物（小羊、小兔、小鹿、小狐狸），用简单几何体组合表现
- 动物在地面随机漫步，走走停停，有自然的待机和行走动画
- 动物避开战斗区域（半径 6 以内）和障碍物位置
- 集成到现有的 EnvironmentManager 更新循环中
- 支持场景清理（dispose）

**Non-Goals:**
- 不与玩家碰撞或交互（纯装饰）
- 不与敌人和子弹交互
- 不添加 UI 元素（不显示动物名称、血条等）
- 不使用外部模型文件（全部用 Three.js 几何体构建）

## Decisions

### 1. 动物架构设计：独立的 ForestAnimal 类
参照 Bird 模式，每个动物实例是 `ForestAnimal` 类，包含：
- `group: THREE.Group` — 根节点
- `update(delta)` — 更新位置和动画
- `dispose()` — 清理资源

### 2. 行为 AI：简单状态机
每种动物有 3 个状态：
- **IDLE**（待机）：停留 2~5 秒，播放待机动画（耳朵晃动、尾巴摇摆、四处张望）
- **WALKING**（行走）：向随机目标点行走 3~8 秒，播放行走动画（四肢交替摆动）
- **FLEEING**（逃离）：当玩家接近到 8 单位以内时，朝反方向小跑 2~3 秒

### 3. 活动范围控制
- 最小半径 6（避开中心战斗区）
- 最大半径 20（不超出场景边界）
- 随机选择目标点时，避开已知障碍物位置（使用预定义的障碍物坐标列表）

### 4. 动物视觉设计（Toon 风格）
沿用项目的 `MeshToonMaterial` + `gradientMap` 风格：

| 动物 | 身体 | 特征 |
|------|------|------|
| 🐑 小羊 | 白色球体组合（蓬松感） | 小耳朵、短腿、小尾巴 |
| 🐰 小兔 | 浅灰身体 | 长耳朵（竖立）、圆尾巴 |
| 🦌 小鹿 | 棕色身体 | 小角（锥体）、长腿 |
| 🦊 小狐狸 | 橙色身体 | 尖耳朵（锥体）、大尾巴 |

### 5. 动物数量
- 每局游戏生成 6~8 只动物
- 动物分布在场景四周，避免扎堆

### 6. 性能考虑
- 每个动物使用约 8~12 个几何体（MeshBasicMaterial），与小鸟复杂度相当
- 总共约 48~96 个额外几何体，对性能影响极小
- 使用 `MeshBasicMaterial`（无光照计算）以保持性能

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|---------|
| 动物走到障碍物内部 | 目标点选择时过滤障碍物坐标区域 |
| 动物太多影响性能 | 限制最多 8 只，使用简单几何体 |
| 动物与敌人重叠 | 动物 Y 轴偏移 0.1~0.3，不与实体碰撞体冲突 |
| 玩家吓跑动物逻辑过于复杂 | 仅在玩家距离 < 8 时触发逃离，保持简单 |
