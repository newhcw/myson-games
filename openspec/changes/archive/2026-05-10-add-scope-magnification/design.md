## Context

当前项目已经实现了基础的倍镜切换功能（toggleScope），可以在狙击枪等武器上开启/关闭倍镜。但现有的实现只做了以下事情：
1. 将 `currentScope.isActive` 设为 true/false
2. 将 `currentScope.magnification` 设为对应的倍率值
3. 武器类型中定义了 `scopeMultiplier` 属性（如狙击枪为 4x）

然而，实际的视觉效果并未实现——玩家开启倍镜后只是视野变窄，没有真正的放大效果。对于儿童游戏来说，真实的倍镜放大体验是一个重要的沉浸式功能。

## Goals / Non-Goals

**Goals:**
- 实现真实的倍镜视觉放大效果，通过动态调整相机 FOV 实现
- 提供平滑的 FOV 过渡动画，避免视野突变带来的不适
- 添加开镜时的暗角效果（vignette），增强瞄准沉浸感
- 支持开镜时准星替换为精确瞄准十字
- 减少开镜时的视角摇晃，提升瞄准稳定性

**Non-Goals:**
- 不实现真正的光学��理模拟（如色散、畸变等复杂效果）
- 不支持热成像、夜视等特殊瞄准镜模式
- 不添加子弹时间（bullet time）或慢动作效果
- 不修改现有的武器数据配置结构（仅扩展功能）

## Decisions

### D1: 使用 FOV 调整实现倍镜放大效果

**决策:** 通过动态调整 Three.js 相机的视场角（FOV）来模拟倍镜放大效果。

**备选方案:**
- A. 移动相机位置（远离玩家）—— 会改变透视关系，不适合
- B. 使用后处理 shader 放大局部画面 —— 复杂度高，性能开销大
- C. 调整 FOV —— 实现简单，性能好，效果直观

**理由:** FOV 调整是实现倍镜放大效果最简洁高效的方式。Three.js 的 PerspectiveCamera 原生支持动态 FOV 修改，切换到更小的 FOV（如从 75° 降到 20°）会使得视野中的物体看起来更大。

### D2: FOV 过渡动画使用线性插值

**决策:** 倍镜开启/关闭时的 FOV 变化采用线性插值（Lerp），过渡时长 200ms。

**备选方案:**
- A. 立即切换 —— 视野突变，造成不适
- B. 缓动（ease-out）—— 需要额外的缓动函数
- C. 线性插值 —— 效果自然，实现简单

**理由:** 线性插值足够平滑，且实现简单。200ms 的过渡时间既能提供流畅的视觉体验，又不会让玩家感觉反应迟钝。

### D3: 暗角效果使用 CSS overlay 实现

**决策:** 使用全屏 CSS overlay 添加暗角效果，而非后处理 shader。

**备选方案:**
- A. 后处理 shader（Post-processing）—— 效果好但实现复杂
- B. CSS radial-gradient overlay —— 简单高效，易于定制

**理由:** CSS 实现足够轻量，且可以轻松调整暗角强度和颜色。对于儿童游戏来说，简洁的实现更易于维护和调试。

### D4: 准星使用 SVG 动态替换

**决策:** 在 HUD 层添加 SVG 准星，通过 CSS 控制显示/隐藏和样式切换。

**备选方案:**
- A. Canvas 绘制准星 —— 需要额外的渲染循环
- B. 图片切换 —— 不够灵活
- C. SVG 动态准星 —— 可缩放、可动画、代码可控

**理由:** SVG 准星可以精确控制线条粗细、颜色、动画效果，与游戏整体的矢量风格一致。

## Risks / Trade-offs

- **[风险] FOV 调整可能影响已有的相机效果** → **缓解:** 在 useCameraEffects 中统一管理 FOV 状态，确保与现有的 sway、recoil 等效果兼容
- **[风险] 快速切换倍镜可能导致动画卡顿** → **缓解:** 使用 requestAnimationFrame 确保动画流畅，添加状态锁防止重复触发
- **[风险] 低端设备上 FOV 变化可能影响性能** → **缓解:** FOV 调整是纯相机属性修改，开销极低；暗角效果使用 CSS 而非 WebGL 后处理
- **[风险] 开镜时视野变窄可能导致晕动症** → **缓解:** 提供可选的暗角效果（可关闭），并限制最大放大倍率为 8x
