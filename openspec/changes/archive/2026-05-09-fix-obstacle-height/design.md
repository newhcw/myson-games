## Context

游戏使用 Three.js 程序化生成所有障碍物模型（无需外部资源）。障碍物配置在 `obstacles.ts` 中定义，创建逻辑在 `createObstacles.ts` 中。碰撞检测通过 `Collision.ts` 的 `Box3.setFromObject()` 自动计算包围盒，适配任何尺寸。

玩家身高 1.6 单位（眼睛高度），当前障碍物高度不合理。目标是将树木和岩石调整到真实比例。

## Goals / Non-Goals

**Goals:**
- 大幅提高树木高度（大树 12-18 单位，中树 7-10 单位）
- 提高岩石高度（1.5-3 单位）
- 适度提高灌木和树桩高度
- 调整树干/树冠比例以适配更高树木
- 保持碰撞检测和伤害检测正常工作

**Non-Goals:**
- 不修改花朵高度（花朵本来就该小巧）
- 不修改阴影/材质/颜色
- 不修改障碍物位置
- 不新增障碍物类型

## Decisions

### D1: 高度倍增策略

**决策**：采用"高度倍增"而非"物理精确"策略。树木高度提升到玩家的 4-11 倍（大树 12-18 单位），岩石提升到 1.5-3 单位。

**理由**：游戏是 Q 版卡通风格，不是写实模拟器。高度需要"看起来像真的"但不必精确。同时避免树木过高遮挡玩家全部视线。

### D2: 树干比例从 0.4 提高到 0.5

**决策**：`createObstacles.ts` 中 `trunkHeight = height * 0.5`（原 0.4）。

**理由**：更高的树如果树干太短会看起来不协调。0.5 的比例让树干和树冠各占一半，视觉上更平衡。

### DE3D: 树冠半径缩放调整

**决策**：树冠锥体半径倍数从 `[1.2, 0.9, 0.6]` 调整为 `[1.0, 0.75, 0.5]`。

**理由**：树变高后，用原来的半径倍数锥体会横向过大。缩小半径让树冠更紧凑，形态更接近真实树木。

### D4: 碰撞检测无需修改

**决策**：不修改 `Collision.ts`。

**理由**：`Box3.setFromObject(mesh)` 自动根据几何体计算包围盒，障碍物变大后自动适配。

## Risks / Trade-offs

| 风险 | 缓解方案 |
|------|---------|
| 大树遮挡玩家视线 | 增大树间距，或保持大树在边缘位置 |
| 场景变拥挤 | 现有障碍物位置不变，只调高度 |
| 阴影异常 | 更大的阴影可能影响性能，需要测试 |
| 树干/树冠比例视觉不佳 | 可进一步调整比例参数 |
