## Context

当前游戏架构中，EnemyManager 在 onMounted 时固定生成 5 个敌人（3 小兵 + 1 精英 + 1 BOSS），击杀后 30 秒原地重生。游戏循环在 FPSGame.vue 的 gameLoop 中驱动，每帧调用 enemyAI.update()。道具系统尚未存在。HUD 通过 Vue 响应式数据渲染，没有波次和 Buff 相关UI。

技术约束：纯前端、无后端、全部 3D 模型代码生成、目标用户为儿童。

## Goals / Non-Goals

**Goals:**
- 实现 10 波递进式敌人配置，每波敌人数量和类型递增
- 波次间歇 5 秒倒计时，可按键提前开始
- 击杀敌人掉落道具（生命药水/弹药补给/双倍伤害），Boss 必掉
- Buff 状态管理（双倍伤害 10秒即时生效）
- HUD 显示当前波次、间歇倒计时、Buff 图标及剩余时间
- 第 10 波通关后显示通关界面

**Non-Goals:**
- 不做多个地图场景（仅当前地图）
- 不做局内升级/商店系统（后续 P1）
- 不做无尽模式（当前为 10 波通关制）
- 不做更多道具 / 武器 / 敌人类型（当前 3 种道具）
- 不做刷新门 3D 视觉效果（后续迭代）
- 不做网络排行榜

## Decisions

### D1: WaveManager 独立为单独模块

**决策**：新建 `game/wave/WaveManager.ts`，不在 FPSGame.vue 中内联实现。

**理由**：FPSGame.vue 已超过 1100 行，继续膨胀会严重影响可维护性。WaveManager 作为独立模块，通过回调函数（onWaveStart/onWaveClear/onAllWavesComplete）与 FPSGame 通信，职责清晰，也便于后续单元测试。

**替代方案**：放在 gameStore 中 → 不推荐，波次逻辑是运行时战斗状态，不是持久化状态。

### D2: 道具掉落概率模型

**决策**：小兵 20%、精英 50%、BOSS 100%（掉落 2 个）。道具类型在 3 种中等概率随机。

**理由**：儿童玩家需要正向反馈，不能太低导致"打了半天什么都没掉"。但不能太高导致屏幕混乱，同屏道具上限设为 8 个。

**替代方案**：固定掉落表 → 后续可做，当前 MVP 用随机即可。

### D3: 道具 3D 模型用简单几何体组合

**决策**：生命药水 = 两个半球 + 圆柱组合成心形；弹药补给 = 立方体；双倍伤害 = 八面体（OctahedronGeometry）。全部使用 MeshStandardMaterial + emissive 自发光。

**理由**：与项目现有风格一致（全部代码生成模型），不需要外部资源。儿童对抽象几何体接受度高，配合发光和旋转即可有足够辨识度。

### D4: Buff 状态放在独立 stores/buffs.ts

**决策**：新建 Pinia store 管理 Buff 状态，不放在 gameStore 中。

**理由**：gameStore 已有 score/kills/health/gameState 等职责，Buff 系统（多个Buff同时存在、各自倒计时、效果叠加）有独立的状态管理需求。独立 store 也方便 HUD 组件订阅。

**替代方案**：放在 WaveManager 中 → 不利于 UI 读取。放在 gameStore → 职责过重。

### E5: 波次数据与逻辑分离

**决策**：波次配置（每波出什么敌人、出几个）用纯数据表 `WAVE_CONFIGS` 定义，WaveManager 读取数据表驱动行为。

**理由**：后续调整难度只需改数据表，不改逻辑。也方便后续做关卡编辑器。

### E6: 波次敌人生成位置

**决策**：从预定义的 4 个角落刷新点（±20, ±20）中随机选 2-3 个点生成敌人。不做 3D 刷新门动画。

**理由**：避免玩家蹲在一个位置防守，增加走动的必要性。角落刷新点简单可靠，后续可做刷新门视觉效果。

### E7: 通关界面为全屏遮罩组件

**决策**：新建 `game/ui/VictoryScreen.vue`，类似现有的 DeathScreen.vue，全全屏遮罩 + 本局数据展示 + 按钮。

**理由**：复用 DeathScreen 的设计模式，保持一致性。

## Risks / Trade-offs

| 风险 | 缓解方案 |
|------|---------|
| 波次难度跳跃太大，儿童玩家挫败 | 前 3 波伤害降低 30%（教学波）；第 3 波才开始出现精英 |
| 道具太多导致视觉混乱 | 同屏道具上限 8 个，超出时移除最早掉落的道具 |
| 波次间歇时玩家无事可做 | 允许间歇时移动 / 换武器 / 拾取上波掉落的道具 |
| 多个 Buff 同时存在时效果不清晰 | HUD 每个 Buff 独立图标 + 彩色倒计时环，颜色编码与道具一致 |
| 计时器叠加导致内存泄漏 | WaveManager 提供 dispose() 方法，FPSGame onUnmounted 时调用 |
| 通关后重新开始状态不完整 | VictoryScreen 的"重新开始"按钮复用现有的 onRestart 逻辑 |
