## Context

当前游戏中的道具掉落系统已经完整实现（PowerUpManager、掉落判定逻辑、拾取检测），但存在两个问题：

1. **掉落概率过低**：Soldier（小兵）仅有 20% 掉落概率，玩家难以获得血量补充
2. **缺乏 UI 提示**：玩家不知道击败敌人会掉落道具，也不知道如何拾取

项目约束：
- 纯 H5 实现，无后端
- 目标用户是儿童，UI 需保持 Q 版卡通风格
- 所有数据本地存储

## Goals / Non-Goals

**Goals:**
- 提高敌人道具掉落率，让玩家更容易获得血量补充
- 添加游戏内 UI 提示，让玩家知晓道具掉落机制
- 不影响现有游戏平衡（双倍伤害Buff时间、弹药补给机制保持不变）

**Non-Goals:**
- 不增加新的道具类型
- 不修改现有道具效果数值（血包恢复 30 点保持不变）
- 不修改敌人 AI 行为

## Decisions

### 1. 掉落率调整方案
- Soldier：20% → 50%
- Elite：保持 50%（已足够）
- Boss：保持 100%

**备选方案考虑：**
- 方案 A（采用）：直接提升掉落概率，简单直接
- 方案 B：添加"首杀必掉落"机制 → 过于复杂，不适合儿童用户理解
- 方案 C：随波次增加掉落率 → 属于难度曲线调整，可后续迭代

### 2. UI 提示方案
- 游戏开始时在屏幕中央显示提示文字，3 秒后淡出
- 道具掉落时播放音效（复用现有音效资源）
- 拾取道具时在 HUD 显示简短提示（如"血包 +30"）

**实现技术方案：**
- 使用 Canvas 2D 叠加层显示文字提示（与现有 DamageFeedback 一致）
- 使用现有的 SoundManager 添加简单音效

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|-----|-----|---------|
| 掉落率提升后道具过多 | 可能影响游戏难度 | 设置同屏上限（MAX_POWERUPS=8），超出后移除最早掉落的 |
| UI 提示过于频繁 | 干扰游戏体验 | 仅在首次游戏时显示提示，后续可关闭 |
| 音效提示过于频繁 | 噪音干扰 | 音效音量适中，且只有拾取时播放 |

## Open Questions

1. **提示是否需要记忆？** — 首次显示后是否需要记住不再显示？（用户反馈后再决定）
2. **是否需要移动端适配？** — 目前游戏支持触屏，提示文字需适配小屏幕
