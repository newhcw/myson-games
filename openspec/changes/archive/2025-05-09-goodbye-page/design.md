## Context

当前项目是一个儿童 3D FPS 游戏，使用 Vue 3 + Vite + Three.js 构建。游戏有主页（Home）、游戏界面（FPSGame）和设置页面（Settings）。当用户点击主页的"离开森林"按钮时，目前没有对应的告别页面，而是直接退出或报错。

本变更需要实现一个温馨的告别页面，提供良好的用户体验。

## Goals / Non-Goals

**Goals:**
- 创建 `/goodbye` 路由和对应的 Goodbye.vue 页面组件
- 实现夜晚森林风格的视觉效果，与主页白天场景形成对比
- 展示本次游戏的统计数据（得分、击杀数、存活时间）
- 提供"再玩一次"和"回家啦"两个交互按钮
- 添加动态效果（萤火虫、星星）增强视觉吸引力

**Non-Goals:**
- 不涉及游戏核心玩法的修改
- 不添加后端服务或 API
- 不修改现有的游戏存档机制（游戏数据读取使用现有 store）

## Decisions

### 1. 页面架构
采用单文��组件（.vue）实现，结构与现有页面（Home.vue、Settings.vue）保持一致。

### 2. 数据获取
使用现有的 Pinia store（gameStore）获取本次游戏的统计数据。如果无存档数据，显示"首次来访"状态。

### 3. 视觉风格
- 背景：夜晚森林渐变（深蓝 → 紫色 → 深绿）
- 动态元素：CSS 动画实现的萤火虫和星星
- 角色：使用 emoji 字符模拟小动物角色（🦊、🦉、🐰）
- 按钮：保持与主页一致的圆角大按钮风格

### 4. 路由配置
在现有路由配置中添加 `/goodbye` 路由，使用懒加载。

## Risks / Trade-offs

[风险] 儿童可能误触退出按钮
→  Mitigation: 提供明确的"再玩一次"选项作为默认推荐

[风险] 无存档时数据显示为空
→  Mitigation: 显示友好的默认文案，如"下次记得多玩一会儿哦"

[风险] 样式与现有页面不一致
→  Mitigation: 复用现有的 CSS 变量（--font-display, --font-body 等）
