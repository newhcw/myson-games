# 修复障碍物悬浮问题

## 问题描述
游戏场景中的障碍物（树木、岩石、灌木、花朵、树桩）悬浮在空中，底部没有接触地面。

## 根因分析
障碍物配置 (`obstacles.ts`) 中的 `pos[1]`（Y 坐标）被设置为模型半高，但各个 `create*Mesh` 函数内部已经将子元素在本地坐标系中向上偏移（使底部对齐 group 原点）。最终 `group.position.y = pos[1]` 导致整个 group 被抬升，障碍物底部离地 pos[1] 高度。

实际上每个 create 函数中，子元素的本地 Y 坐标已经处理了"从 0 向上排列"的逻辑（树干底部在本地 Y=0，树干中心在本地 Y=trunkHeight/2）。所以 group 应该放在 Y=0（地面高度），而不是 pos[1]。

## 修复方案
将所有障碍物的 `pos[1]` 改为 0，即障碍物 group 直接放在地面上，内部元素自动从地面向上排列。
