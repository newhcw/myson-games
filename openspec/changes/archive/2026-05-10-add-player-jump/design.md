## Context

当前 `usePlayerMovement.ts` 已有基础的跳跃逻辑：
- 重力 `gravity = -9.8`
- 跳跃初速度 `jumpVelocity = 5`
- 跳跃高度约 1.27m（= v²/2g）

但存在关键问题：
1. **跳跃高度不够** — 树桩高 1.2-1.5m，灌木高 1.5-2.5m，玩家跳不过任何障碍物
2. **碰撞检测只考虑水平方向** — `CollisionDetector.checkCollision()` 检查 `playerBox` 是固定高度（y: -1 ~ +0.5），跳跃到障碍物上方时仍会被检测到碰撞
3. **空格键未绑定跳跃** — 键盘处理中没有映射空格到 jump()

## Goals / Non-Goals

**Goals:**
- 玩家能跳跃穿越矮障碍物（树桩、小灌木、小岩石，高度 ≤ 2m）
- 碰撞检测支持 Y 轴：玩家在空中时不会与矮障碍物水平碰撞
- 可变跳跃高度：按住跳跃键跳得更高
- 落地检测和相机反馈
- 触屏虚拟摇杆添加跳跃按钮

**Non-Goals:**
- 二段跳或其他高级跳跃机制
- 攀爬/翻越系统
- 跳跃攻击
- 掉落伤害

## Decisions

### 决策 1：跳跃参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 重力 | -14.7 (1.5x 当前) | 更快下落，手感更敏捷 |
| 跳跃初速度 | 8 | 搭配重力 => 最高点 ~2.18m |
| 最小跳跃速度 | 5 | 轻按时的初速度 => ~0.85m |
| 跳跃蓄力窗口 | 200ms | 按住前 200ms 内持续加速到最大 |

相比当前（gravity=9.8, velocity=5），新参数使最大跳跃高度从 1.27m → 2.18m，足够越过树桩 (1.5m) 和小灌木 (1.8m)。

### 决策 2：空中碰撞检测策略

**方案**：跳跃时禁用水平碰撞检测的 Y 轴矮障碍物阻挡。

在 `CollisionDetector.checkCollision()` 中添加空中模式参数：
- 当玩家在空中且 `playerBox` 底部高于障碍物顶部时，不判定为碰撞
- 需要为每个障碍物保存顶部高度（从 mesh 的 bounding box 获取）
- 落地时（player 底部 <= 障碍物顶部 + 容差）视为站在障碍物上

**替代方案考虑**：
- 射线检测：精度更高但计算量大，且与现有 Box3 碰撞系统不兼容
- 每个障碍物添加平台 collider：实现复杂，且玩家不应该能站在大树/灌木上

**结论**：采用 Box3 顶部高度比较法，实现简单且效果足够。

### 决策 3：可变跳跃高度实现

使用「蓄力时间窗口」模式：
1. 玩家按下跳跃键时，开始记录蓄力时间
2. 在 `update()` 循环中，如果跳跃键持续按住且蓄力 < 200ms，持续增加跳跃速度
3. 释放跳跃键或蓄力超时后，固定速度进入上升阶段
4. 上升过程中不再受输入影响

这样玩家轻按空格跳得低（约 0.85m），长按跳得高（约 2.18m）。

### 决策 4：落地检测和相机反馈

- 落地时：如果下落速度 > 阈值（-5），触发相机小幅度震动
- 起跳时：相机轻微下沉再复位（模拟腿部发力）
- 使用现有的 `useCameraEffects` 中的 `startCameraShake()`

## Risks / Trade-offs

- **[风险] 跳跃穿越障碍物导致看不到背后敌人** → 跳跃高度有限（~2m），障碍物前方视线本就会被遮挡，影响可控
- **[风险] 在障碍物顶上「着陆」的怪异行为** → 落地时强制滑落到地面，不实现站障碍物逻辑（保持简单）
- **[风险] 触屏跳跃按钮误触** → 跳跃按钮做大但放在合理位置，与射击按钮分开
- **[风险] 修改重力参数影响现有手感** → 目前玩家只有地面移动，重力只影响跳跃，无其他依赖
