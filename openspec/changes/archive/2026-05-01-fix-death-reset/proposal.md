# 修复死亡/重置/HUD 显示问题

## 背景

当前游戏存在三个用户可观察的 bug：

1. **玩家死亡后游戏仍在运行** — 死亡界面弹出后，敌人 AI 继续移动、火箭和敌人投射物仍在飞行、键盘输入仍在处理，玩家实际上还能"操作"一个已死亡的角色
2. **重新开始按钮没有正确重置敌人** — `onRestart()` 只重置了 gameStore 和玩家位置，没有清理敌人系统、火箭管理器、投射物管理器和武器弹药状态，导致重启后敌人状态残留
3. **大量敌人时血条重叠** — `EnemyHealthBar` 的每个血条独立计算屏幕坐标，当多个敌人在屏幕上的投影位置接近时，血条 DOM 元素直接重叠，视觉混乱

## 范围

- **死亡处理**：死亡时暂停游戏循环，冻结所有游戏活动
- **重置机制**：重新开始时完整清理并重新初始化敌人、火箭、投射物、弹药
- **血条显示**：重叠时自动调整布局，避免遮挡
- 纯前端改动，不涉及后端

## 目标

- 死亡时敌人停止移动、火箭和投射物停止飞行
- 点击重新开始后敌人回到初始位置、弹药满、所有飞行物清除
- 多敌人时血条不重叠，信息清晰可读
